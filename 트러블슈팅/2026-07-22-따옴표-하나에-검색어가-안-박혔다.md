# 2026-07-22 따옴표 하나 때문에 검색어가 제목에 안 박혔다

> 태그: #트러블슈팅 #국제화 #문자열

## 무엇을 / 왜
검색 결과 제목에 검색어를 넣어 `'티셔츠' 검색 결과`처럼 보여주려 했다. 그런데 화면엔 검색어 대신
`{query}`라는 글자가 그대로 떴다. 분명 자리표시자(placeholder)에 검색어를 넣어줬는데도 안 바뀌었다.

## 어떻게 풀었나
다국어 문구는 보통 `'{query}' 검색 결과`처럼 써두고, 실행할 때 `{query}` 자리에 실제 값을 끼워 넣는다.
그런데 이 문구 방식(ICU MessageFormat)에서는 **작은따옴표(')가 특수한 뜻**을 가진다. 따옴표로 감싼
부분은 "여기 안쪽은 문법으로 해석하지 말고 글자 그대로 둬라"는 이스케이프 신호다.

그래서 `'{query}'`는 "따옴표로 감싼 리터럴 `{query}`"로 해석되어, 자리표시자가 아니라 그냥 글자
`{query}`가 되어버렸다. 값이 안 끼워지는 게 당연했다.

해결은 **특수한 뜻이 없는 따옴표로 바꾸는 것**이었다. 작은따옴표(') 대신 둥근 따옴표(' ')를 쓰면
이스케이프로 안 걸려서 자리표시자가 정상 치환된다. 보기에도 더 깔끔했다.

## 코드로 보기 (개념 재현)
```
// ICU MessageFormat: 작은따옴표(')는 "이스케이프" - 감싼 부분을 글자 그대로 둔다.

"'{query}' 검색 결과"      // ✗ '{query}' 가 리터럴로 굳어 -> 화면: "{query} 검색 결과"
"''{query}'' 검색 결과"    // △ 따옴표를 두 번('')써야 리터럴 따옴표가 됨 - 되지만 헷갈림
"‘{query}’ 검색 결과"      // ✓ 둥근 따옴표(U+2018/2019)는 특수문자가 아님 -> 정상 치환

// format({ query: "티셔츠" })  =>  "‘티셔츠’ 검색 결과"
```

## 💡 배운 것
- **다국어 문구에서 따옴표·중괄호 같은 문자는 "그냥 글자"가 아니라 문법일 수 있다.** 특히 작은따옴표는
  여러 문구 포맷에서 이스케이프 기호로 쓰인다.
- **자리표시자가 통째로 안 박히면, 값 문제보다 "문구 자체가 그 자리를 자리표시자로 안 본 것"을 의심한다.**
- 화면에 template이 글자 그대로(`{query}`) 뜨면, 데이터가 아니라 **문구 해석 단계**를 먼저 본다.

## 소감
값을 아무리 잘 넘겨도 소용없고, 문구 안의 따옴표 하나가 범인이었다. "왜 값이 안 들어가지"에서 "왜
자리표시자로 안 보지"로 질문을 바꾸니 원인이 보였다. 작은 문법 규칙 하나가 화면을 망칠 수 있다는 걸
배웠다.
