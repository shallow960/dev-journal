# 2026-07-22 주문은 되는데 재고가 안 줄고 있었다

> 태그: #트러블슈팅 #동시성 #재고 #데이터정합

## 무엇을 / 왜
재고 관련 화면을 손보다가 "주문하면 재고가 진짜 줄긴 하나?"가 궁금해서 뒷단을 열어봤다. 그런데 재고를
줄이는 부분이 **"아무 일도 안 하는" 임시 코드**로 막혀 있었다. 그래서 재고가 0이어도 계속 팔리고
(오버셀), 품절 표시도 자동으로 안 바뀌던 상태였다.

## 어떻게 풀었나
처음엔 "재고를 읽어서 충분한지 보고, 그 다음 줄이면 되겠지" 했다. 그런데 이렇게 하면 두 요청이 거의
동시에 들어올 때 어긋난다. 둘 다 "재고 1개 남음"을 읽고, 둘 다 "충분하네" 하고 각자 줄여서, 재고가
음수가 되거나 없는 물건을 두 사람에게 파는 일이 생긴다(경합, race condition).

그래서 **읽기와 줄이기를 하나의 명령으로 합쳤다.**

- "남은 재고가 주문 수량 이상일 때만, 그만큼 줄인다"를 조건이 붙은 **한 번의 UPDATE**로 처리한다.
  데이터베이스가 그 줄을 처리하는 동안 해당 행을 잠그기 때문에, 동시 요청은 줄을 서서 하나씩 처리되고,
  모자라면 아예 안 줄고 주문을 막는다.
- 재고가 0이 되면 품절 표시도 **같은 UPDATE에서 함께** 켠다.
- 취소/환불로 주문이 끝나면 줄였던 만큼 도로 더해 복원한다.
- 함정 하나: 테스트에선 실제로 없는 임시 상품 번호로 주문을 넣는데, "UPDATE가 0줄 바꿈 = 재고 부족"
  으로 단정하면 그런 테스트가 다 실패한다. 그래서 **"상품이 실제로 있는데 모자란 것"과 "그 상품이 아예
  없는 것"을 구분**해서, 실제 상품일 때만 주문을 막게 했다.

검증: 재고 2개짜리를 2개 주문 → 재고 0 + 품절, 거기서 1개 더 → 차단, 취소 → 다시 2개 + 품절 해제.
회귀 테스트로 남겼다.

## 코드로 보기 (개념 재현)
```java
// 핵심: "확인 + 차감"을 한 번의 조건부 UPDATE로. 읽고 나서 줄이는 2단계로 하지 않는다.
int updated = db.update("""
    update product
       set stock = stock - :qty,
           sold_out = (stock - :qty = 0)   -- 0이 되면 품절도 같은 자리에서 켠다
     where id = :id
       and stock >= :qty                   -- ★모자라면 이 조건이 안 걸려 0줄 갱신
    """, id, qty);

if (updated == 0) {
    // 0줄 = (a) 재고 부족  또는  (b) 그런 상품이 없음 - 둘을 구분해야 한다
    boolean exists = db.exists("select 1 from product where id = :id", id);
    if (exists) {
        throw new SoldOutException();       // 실제 상품인데 모자람 -> 주문 차단
    }
    // 상품이 없으면(테스트용 임의 번호 등) 관용적으로 넘어간다
}

// 취소/환불 시 복원 - 줄였던 만큼 도로 더하고 품절 해제
db.update("update product set stock = stock + :qty, sold_out = false where id = :id", id, qty);
```

## 💡 배운 것
- **"읽고 나서 바꾸기"는 동시에 들어오면 어긋난다.** 재고·잔액처럼 여러 요청이 다투는 값은 "확인과
  변경을 한 번의 조건부 UPDATE로" 묶어야 안전하다. 조건(`stock >= qty`)이 곧 잠금이자 방어선이 된다.
- **"아무 일도 안 하는" 임시 코드가 제일 조용한 버그다.** 나중에 채우려고 비워둔 자리는 티가 안 나서,
  정작 안 도는 걸 한참 모른다.
- **0줄 갱신에도 이유가 여럿이다.** "안 바뀜"을 한 가지 원인으로 단정하면 엉뚱한 곳에서 터진다.

## 소감
재고쯤이야 잔액에서 빼면 되는 줄 알았는데, 두 사람이 동시에 마지막 하나를 사는 그림을 떠올리고 나서야
왜 한 번에 묶어야 하는지 이해됐다. 잘 돌던 것도 "이거 진짜 도나?"를 한 번 열어보는 게 중요하다는 걸
다시 느꼈다.
