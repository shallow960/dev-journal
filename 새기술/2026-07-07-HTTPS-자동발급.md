# 2026-07-07 리버스 프록시로 HTTPS 자동 발급·갱신 붙이기

> 태그: #새기술 #인프라 #HTTPS #배포

## 무엇을 / 왜
도메인을 서버에 연결했는데 아직 평문(http)이었다. 로그인·결제가 오가는 사이트라 HTTPS는 필수인데, 인증서를 손으로 발급받고 90일마다 갱신하는 건 잊어버리기 쉽고 번거롭다. 사람 손 안 타는 자동화가 필요했다.

## 어떻게 풀었나
서버 앞단에 리버스 프록시를 한 층 세웠다. 이 프록시가 세 가지를 알아서 한다 - (1) 무료 공인 인증서를 자동으로 발급받고 (2) 만료 전에 스스로 갱신하고 (3) 평문(http) 요청은 암호화 연결(https)로 리다이렉트한다. 앱은 그대로 두고 프록시만 앞에 붙이는 구조라, 돌아가던 서비스를 멈추지 않고 적용했다.

자동 발급이 되는 조건은 딱 세 가지였다 - 도메인이 이 서버를 가리키고(DNS 완료), 인증 검증용 포트(80/443)가 열려 있고, 서버에 자동 발급 프로토콜(ACME)을 처리할 프록시가 떠 있으면 된다. 이 셋이 맞으니 프록시 컨테이너가 뜬 지 몇 초 만에 인증서가 발급됐다.

관리형 로드밸런서(월 유료)를 앞에 두는 대안도 있었지만, 단일 서버엔 프록시 방식이 무료이고 더 간단했다.

## 💡 배운 것
- HTTPS 인증서는 이제 "손으로 받아 파일을 넣는" 게 아니라, 자동 발급 프로토콜(ACME)을 지원하는 도구에 맡기면 발급·갱신을 통째로 자동화할 수 있다. 갱신을 잊어 사이트 인증서가 만료되는 대표적 사고를 원천 차단한다.
- 자동 발급이 되려면 "도메인이 이 서버를 가리킴 + 검증용 포트 열림 + ACME 처리 주체 존재" 세 조건이 필요하다. 안 될 땐 이 셋 중 뭐가 빠졌는지 보면 원인이 좁혀진다.
- 기존 앱을 안 건드리고 앞에 프록시 한 층을 얹는 방식은, 무중단으로 기능(HTTPS·리다이렉트)을 추가하는 깔끔한 패턴이다.

## 코드로 보기 (구조 골격)
프록시 설정이 얼마나 단순한지, 자동 HTTPS의 핵심이 사실상 "도메인 선언"뿐임을 보인다.

```
# 리버스 프록시 설정 (개념) - 이 몇 줄이 인증서 발급·갱신·리다이렉트를 다 처리한다.
# 도메인 두 개를 선언하면, 프록시가 알아서 인증서를 받아 https 로 서비스한다.
example.com, www.example.com {
    # 뒤에 있는 앱(로컬 포트 3000)으로 요청을 전달
    reverse_proxy localhost:3000
}

# 프록시가 자동으로 처리하는 일:
#  1) 기동/최초 요청 시 공인 CA 에 인증서 요청(ACME) -> 도메인 소유 검증 -> 발급
#  2) 만료 30일쯤 전 자동 갱신 (사람 개입 0)
#  3) http:// 로 들어오면 https:// 로 리다이렉트(308)
```

## 소감
예전엔 인증서라 하면 발급받고, 파일 옮기고, 갱신 알림 맞추고... 잊으면 사고 나는 번거로운 일이었다. 도구에 맡기니 설정 몇 줄로 끝났고, "사람이 잊어버릴 수 있는 일은 자동화로 없앤다"는 걸 다시 체감했다. 유료 대안 대신 무료로 해결한 것도 좋았다.
