# 2026-07-13 오픈소스를 채택할까, 참고할까

> 태그: #개선 #판단 #아키텍처 #제품

## 무엇을 / 왜
"검증된 오픈소스 커머스가 잘 만들어져 있으니 우리도 그걸 쓰자"는 얘기를 들었다. 처음엔 그럴듯했다 -
이미 완성된 몰에 화면 구성도 유연하고 결제까지 다 있으니까, 안 쓸 이유가 없어 보였다.

그런데 바로 가져다 쓰기 전에 "이게 우리가 만드는 거랑 맞나"부터 따져 보기로 했다. 잘못 가져오면
지금까지 만든 걸 버려야 하고, 관리할 기술 스택이 하나 더 늘고, 방향도 어긋날 것 같았다.

## 어떻게 풀었나
도입을 정해 놓고 시작하지 않고, 먼저 우리 제품이랑 나란히 놓고 비교해 봤다.

- **원하던 게 이미 있는지부터 봤다.** "유연한 화면"이랑 "돌아가는 몰"은 우리 제품에 이미 있었고,
  없는 것(검색, 해외 결제)은 그 오픈소스랑 상관없이 따로 붙일 수 있는 기능들이었다.
- **제일 큰 차이가 뭔지 짚어 봤다.** 그 오픈소스는 근본이 "몰 하나를 세우는 도구"였다. 반면 우리
  제품은 "여러 판매자에게 각자의 몰을 파는 구독형(멀티테넌트) 플랫폼"이다. 전자를 후자처럼 쓰려면
  고객 수만큼 복제본을 만들거나 핵심을 크게 뜯어고쳐야 했다 - 우리 제품의 제일 중요한 차별점이랑
  구조가 안 맞는 거다.
- **그래서 "채택"이 아니라 "참고"로 정했다.** 통째로 가져오는 대신, 잘 만들어진 그 코드에서 설계랑
  보안 로직만 우리 스택에 맞게 옮겨 오기로 하고, 이 판단을 대표님께 설명할 문서로 정리했다.

## 💡 배운 것
- "좋은 거 있으니 쓰자"는 말을 처음엔 그냥 따라야 하는 줄 알았다. 그런데 우리 제품이랑 하나씩
  대조해 보니, 그 말의 진짜 뜻은 "이런 기능이나 품질을 원한다"였다. 시키는 대로 바로 하기 전에
  "왜 이걸 원하실까"를 한 번 되물어 보는 게 이렇게 중요한 줄 몰랐다.
- 기성품을 쓸지 직접 만들지가, 뭐가 기술적으로 더 좋냐의 문제가 아니라 "우리가 뭘 파는 회사냐"에서
  갈린다는 게 신기했다. 몰 하나만 세우는 거면 기성품이 빠른 게 맞는데, 우리처럼 여러 몰을 찍어내는
  구조에선 아무리 좋은 기성품도 안 맞을 수 있다는 걸 이번에 처음 알았다.
- "참고"랑 "채택"이 이렇게 다른 결정인 줄 몰랐다. 코드에서 배워서 가져오는 거랑, 통째로 우리 스택에
  들이는 건 완전히 다른 얘기였다. 아이디어는 맘껏 가져오되 실제로 돌아가는 건 우리 걸로 두는 게
  나을 때가 있다는 걸 알았다.

## 코드로 보기 (구조 골격)
왜 "단일 몰 도구"를 "멀티테넌트 플랫폼"으로 쓰기 어려운지를, 데이터 접근의 뼈대로 재현한다.

```ts
// 단일 사이트 도구: "이 설치본 = 하나의 몰"이라는 전제.
// 조회에 소유 조직 개념이 없다. 인스턴스 자체가 곧 그 몰이다.
function listProductsSingleSite() {
  return db.query("select * from products"); // 전 상품 = 이 몰의 상품
}

// 멀티테넌트 플랫폼: 한 시스템이 수많은 몰을 격리해 담는다.
// 모든 조회가 "어느 몰이냐(tenant)"로 스코프돼야 안 섞인다.
function listProductsMultiTenant(tenantId: string) {
  return db.query("select * from products where tenant_id = $1", [tenantId]);
}

// 단일 사이트 도구로 멀티테넌트를 흉내내려면,
// 결국 고객 수만큼 인스턴스를 복제해야 한다 - 운영비가 고객 수에 비례해 늘어난다.
//   customerA -> 설치본 A (DB A)
//   customerB -> 설치본 B (DB B)  ...
// 반면 멀티테넌트는 코드/DB 하나에서 tenant 로 격리한다. 이게 구조적 차이다.
```

## 소감
"좋은 걸 가져다 쓰자"는 말에 바로 손이 가지 않고, "그게 우리가 만드는 물건이랑 같은 종류인가"를
먼저 따져본 게 이번엔 다행이었다. 코드를 얼마나 잘 짜느냐보다, 뭘 만들고 뭘 안 만들지 정하는 판단이
결과를 더 크게 바꾼다는 걸 조금은 알 것 같다. 좋은 도구를 안 쓰기로 하는 것도 결정이더라.
