# 2026-07-13 단어 순서에 걸리지 않는 검색

> 태그: #개선 #검색 #UX

## 무엇을 / 왜
상품 검색이 이미 붙어 있었는데, 실제로 써 보니 **단어 순서가 바뀌면 결과가 하나도 안 나왔다.**

- "코튼 티셔츠"로 검색 -> 나옴
- "티셔츠 코튼"으로 검색 -> 0건

같은 상품을 찾는 건데 단어를 놓은 순서가 다르다는 이유로 아무것도 못 찾는 것이다. 사용자는 단어를
가나다순이나 정해진 순서로 치지 않는데, 검색은 그걸 요구하고 있었다.

원인은 검색어를 **하나의 덩어리로 부분 문자열 매칭**했기 때문이다. 검색어 전체가 상품명 안에 그
순서대로 연속으로 들어 있어야만 걸렸다.

## 어떻게 풀었나
검색어를 통짜로 다루던 걸 **단어 단위로 쪼갰다.**

- **공백으로 단어를 나누고, 각 단어를 전부 포함(AND)하는 상품을 찾게 했다.** "티셔츠 코튼"이면
  "티셔츠"랑 "코튼"을 따로 찾아서 둘 다 들어 있는 상품을 보여 준다. 이러면 **순서랑 상관없이**
  걸리고, 각 단어는 여전히 부분 매칭이라 "티셔"처럼 일부만 쳐도 나온다.
- **단어를 다 포함해야 나오게(AND) 유지했다.** 서로 다른 상품에 있는 단어를 섞어서 치면(한 상품엔
  둘 다 없는 경우) 0건이 나온다 - 모든 단어가 다 들어 있는 것만 결과라서 엉뚱한 게 섞이지 않는다.
- 검색어에 섞인 특수문자가 엉뚱하게 와일드카드처럼 동작하지 않도록 처리하고, 한 번에 쪼개는 단어
  수에 상한도 뒀다.

고친 뒤 "티셔츠 코튼"(역순)이 정상적으로 걸리고, 서로 다른 상품 단어를 섞으면 0건, 단일/부분/빈
검색도 그대로 동작하는 걸 확인했다.

## 💡 배운 것
- 사용자가 단어를 순서대로 안 친다는 걸 당연하게 생각 못 했다. 검색은 입력을 똑같이 재현하는 게
  아니라 "뭘 찾고 싶은지"를 맞히는 건데, 나는 순서까지 맞아야 걸리게 만들어 놨던 거다. 구현이
  쉬운 방식이 사용자한테도 편한 건 아니라는 걸 알았다.
- 검색엔진 같은 큰 도구를 붙여야 하나 고민했는데, 검색어를 단어로 쪼개서 AND로 거는 작은 변경만으로
  "안 나오던 게 나오는" 큰 차이가 났다. 큰 도구부터 들이기 전에 값싼 개선을 먼저 해 보는 게
  낫구나 싶었다.
- 개선하면서 "이건 아직 안 됨(관련도 정렬, 오타 허용)"까지 같이 적어 두니, 다음에 뭘 하면 될지가
  분명해졌다. 다 못 풀어도 어디까지 풀었는지 적어 두는 게 생각보다 중요하다는 걸 느꼈다.

## 코드로 보기 (구조 골격)
"통짜 부분 문자열(순서 의존)"과 "토큰 AND(순서 무관)"의 차이를 재현한다.

```ts
// 이전: 검색어 전체를 하나의 부분 문자열로 매칭 -> 단어 순서에 걸린다.
function searchWhole(text: string, q: string) {
  return text.toLowerCase().includes(q.toLowerCase());
}

// 이후: 공백으로 토큰을 나눠 각 토큰을 AND 로 매칭 -> 순서 무관 + 부분 매칭 유지.
function searchTokens(text: string, q: string) {
  const haystack = text.toLowerCase();
  const tokens = q.toLowerCase().split(/\s+/).filter(Boolean);
  return tokens.every((t) => haystack.includes(t)); // 모든 토큰이 들어 있어야 매칭(AND)
}

const name = "코튼 티셔츠";
searchWhole(name, "티셔츠 코튼");  // false - 역순이라 통짜 매칭 실패
searchTokens(name, "티셔츠 코튼"); // true  - 두 토큰이 다 있으니 순서 무관하게 매칭
searchTokens(name, "코튼 후드");   // false - "후드"가 없으니 AND 불만족(정확도 유지)
```

## 소감
"검색이 있다"와 "검색이 쓸 만하다"가 다르다는 걸 이번에 느꼈다. 이미 붙어 있는 기능이라도 실제
손님처럼 몇 번 쳐 보니 금세 아쉬운 게 드러났다. 가장 큰 개선이 꼭 가장 큰 도구에서 오는 건
아니더라 - 단어를 쪼개는 한 줄 남짓한 생각이 체감을 제일 크게 바꿨다.
